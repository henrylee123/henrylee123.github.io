mysql 随笔

Posted on 2019-12-02 Edited on 2019-12-05 In 数据库

mysql 中 myisam 与 innode 的区别

  1. innodb 支持事务，MyISAM 不支持
  2. innodb 支持行级锁，MyISAM 支持表级锁
  3. Innode 支持 MVCC，MyISAM 不支持
  4. Innode 支持外键，MyISAM 不支持
  5. Innode 不支持全文索引，MyISAM 支持

MVCC 与 乐观锁

多个用户/线程/进程对数据库进行操作时都会出现三种冲入情况

  1. 读 - 读：没啥冲突
  2. 读 - 写：有隔离性问题，脏读（读到未提交数据），幻读（读已提交 insert），不可重复读 （读已提交 update delete）
  3. 写 - 写：丢失更新

MVCC 是用来解决 读写 问题的一种手段。为每个写操作保存一个版本。读操作读到最新的版本数据。

乐观锁是写写 问题解决手段，在 commit 数据之前看看有没有新 commit 已经抢先提交，若是有则放弃本次 commit。

Mysql 的事务控制

Mysql 有四种级别的事务隔离

READ-UNCOMMITTED

读未提交，一般不用。不怎么加锁。以下读写问题都会有

  • 脏读
时间点 用户A 用户B
1 读 X 表 Y 行数据
2 X 表 Y 行 update 一条数据
3 rollback
4 读 X 表 Y 行数据

1、4 两次 A 用户读到的数据不一致

  • 幻读
时间点 用户A 用户B
1 读 count(X) 表的数据
2 X 表 insert 或 delete 一条数据
3 commit
4 读 count(X) 行的数据

1、4 两次 A 用户读到的数据 count num 不一致

  • 不可重复读
时间点 用户A 用户B
1 读 X 表 Y 行数据
2 X 表 Y 行 update 一条数据
3 commit
4 读 X 表 Y 行数据

1、4 两次 A 用户读到的数据不一致

READ-COMMITED

读已提交，读数据不加锁。修改、写、删除数据加锁。所以会遇到和脏读和不可重复读。

REPEATABLE-READ

可重复读，通过 MVCC 版本控制解决了 不可重复读 的问题。

幻读怎么解决呢？当前事务隔离级别采用了 Next-Key 锁。

  • Next-Key

next-key 锁是行锁 与 GAP 锁 的合并，行锁不用说了，GAP 锁又名 GAP 间隙锁。

  • GAP 锁

GAP 能锁住表级的新增。在表有索引的情况下锁住扫描过的数据。如 where a > 2 and a < 8， 就锁住[2, 8]区间不给 insert 新数据

在这个级别的事务隔离上，可以对 幻读 有一定层度的预防

详情可以看看这篇个详细的 blog :博客 mysql 加锁分析

SERIALIZABLE

完全的锁住

读未提交 V V V
读已提交 X V V
可重复读 X X V
序列化 X X X

linux用户与用户组

Posted on 2019-12-02 Edited on 2019-12-03

用户与组的概念

  1. linux 多用户，多任务的特性

Linux 是一个真实的、完整的多用户多任务操作系统。可以创建多个用户、每个用户在同一时间内可以进行自己的操作完成自己的任务。

用户之间通过权限管理，用户通过不同权限完成不同的任务。实现了多用户多任务的运行机制。

  1. linux 下用户角色

在 linux 下用户根据角色定义，具体分为三种角色：

  • 超级管理员：拥有对系统的最高管理权限，默认为 root 用户
  • 普通管理员：只能对自己目录下的文件进行访问与修改
  • 虚拟用户：不能登录系统，用来运行 web 程序

nginx、noboby 用户在 linux 里面就是一个不能用账号登录的虚拟用户。为了防止程序出现安全问题。不会被黑客获取 root 权限。

  1. linux 用户和组的概念

使用一种系统资源时，须向系统申请一个用户。一个用户组是权限的集合。将用户分组是 linux 系统对用户进行管理及控制方文权限的一种手段。通过定义用户组很大程度上简化而来管理工作。

  1. 用户和组的关系

一个用户组可以拥有多个用户
一个用户可以存在多个用户组中

用户与组的文件概述

  1. 用户和组先关的配置文件
  • /etc/passwd
  • /etc/shadow
  • /etc/group

用户与组的相关命令

  • useradd/adduser
  • userdel
  • usermod
  • groupadd/addgroup
  • groupdel
  • groupmod

buffer cache 的区别

Posted on 2019-11-28 Edited on 2019-12-01 In 操作系统

cache

cache 主要作用缓解 cpu 与普通内存的速度差异。把频繁的寻址地址缓存在 cache 能够优化系统性能。

cache 有三级，一级比一级快。

buffer

buffer 一定程度上减少对 i/o 的设备访问次数。 比如，写入一个文件到磁盘中的时候，会先存一部分在 buffer 里，攒到一定量的时候再写回磁盘中。减少对此盘的访问次数

  2. free
  3. top (进程排序 MEM %)
