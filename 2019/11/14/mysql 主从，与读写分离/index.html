mysql 主从、读写分离

Posted on 2019-11-14 Edited on 2019-12-05 In 数据库

随便写写 mysql 的主从等原理…

mysql 主从，与读写分离

  • mysql 主从的复制原理级流程

基本原理流程就是三个线程之间的关联

主：binlog 线程记录下所有有意义的语句，放入上的 binlog 中
从：io 线程在使用 start slave 后 从 master 拉取 binlog 内容 放进自己的 relay log 中
从：sql 执行线程执行 relay log 中的语句

读写分离

读写分离采用一主多从的方式，数据库的业务一般读多写少。读锁是共享锁，写锁是排它锁。大并发量下，对数据库会造成很大压力。

为什么要做读写分离

随着业务发展，数据量越来越多。度成为系统发展的瓶颈，这时候，应该先引入缓存，memcache，redis。原因是缓存的开发复杂度远远小于读写分离。

当缓存也成为了系统瓶颈的时候，我们考虑数据库分库分表，读写分离的操作。

方案

数据库 master 负责读写数据， 从机负责读数据

数据库主机通过 binlog 把数据同步给从机，每台服务器都有全量的数据。binlog 为 mysql 的二进制日志，用于记录用户对数据库操作的 sql 语句信息 可以使用 mysqlbin 命令查看二进制日志的内容。
